## 왜 직접 만드는가

네이버 블로그나 벨로그 같은 플랫폼도 좋지만, 다음 이유로 직접 만들기로 했다:

1. **수식 지원** — 논문 리뷰에 LaTeX 수식이 필수인데, 직접 만들면 KaTeX로 완벽하게 제어 가능
2. **포트폴리오** — 블로그 자체가 하나의 프로젝트 결과물
3. **소유권** — 글이 전부 내 git 저장소에 마크다운으로 남는다

## 기술 스택

| 역할 | 선택 | 이유 |
|------|------|------|
| 프레임워크 | Astro | 콘텐츠 사이트 특화, 빠른 정적 빌드 |
| 호스팅 | GitHub Pages | 무료, git push만으로 배포 |
| 글 관리 | Sveltia CMS | 웹 관리자 페이지에서 작성 → repo에 자동 커밋 |
| 댓글 | giscus | GitHub Discussions 기반, 서버 불필요 |
